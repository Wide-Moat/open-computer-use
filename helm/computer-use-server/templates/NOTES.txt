{{- if .Values.dind.kataInit.enabled }}

Kata Containers runtime is enabled (dind.kataInit.enabled=true).
  - The dind container runs privileged — this is expected and SAFE under Kata
    (capabilities are confined to the microVM, not the host).
  - Confirm the RuntimeClass exists:  kubectl get runtimeclass {{ default "kata-qemu" .Values.orchestrator.runtimeClassName }}
  - The orchestrator namespace must allow privileged pods. Label it:
      kubectl label ns {{ .Release.Namespace }} pod-security.kubernetes.io/enforce=privileged --overwrite
  - Setup guide: docs/kata-runtime.md
{{- else if hasPrefix "kata" (default "" .Values.orchestrator.runtimeClassName) }}

WARNING: runtimeClassName looks like a Kata RuntimeClass but dind.kataInit.enabled
is false. dockerd will likely fail (overlay2 cannot mount on the virtio-fs guest
root). Set dind.kataInit.enabled=true and dind.privileged=true — see docs/kata-runtime.md.
{{- end }}
